## How posts work

Each post is a single `.mdx` file in `content/blog/`. The frontmatter at the top of the file controls the title, description, date, tags, and an optional cover image.

```yaml
---
title: "Your post title"
description: "One-line teaser for the index page."
date: "2026-05-11"
tags: ["essay", "ai"]
cover: "/blog/your-cover.png"
draft: false
---
```

Set `draft: true` to keep a post out of the index and build while you work on it.

## What you can write

Anything that isn't worth a full case study but is worth saying once:

- Short essays on what you're building and why
- Notes from talks, books, conferences
- Behind-the-scenes for projects already in the portfolio
- Half-formed ideas you want to think out loud about

Markdown works as expected — **bold**, *italic*, [links](https://willschulz.me), inline `code`, fenced blocks, lists, blockquotes, and headings.
